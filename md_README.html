PointCloud
Identify huamans in a point cloud.
Cloud Point README

Introduction:

This small poject got me my first job interview outside of academia. I'm posting it here as is, with minimum modification necessary to respect the confidentiality of the hiring company.

The objective of this project is, given data from a single video frame, to identify players in a basketball game, compute their positions on the field and classify them into players belonging to two teams (TeamA and TeamB) and the Referees.

General Code Description:

This code has been developed and tested on a machine running ubuntu 16.04, but should work on any linux distribution with gcc instaled.

Code structure:

The code is structured into the following sub-directories:

  • include Contains all header files.
  • src Contains all source files that are to be compiled into static libraries.
  • utils Contains all source files that are to be compiled into executables.
  • doc Contains documentation files.
  • share Contains miscallaneous files such as inputs to the code.

After comiling, the following directories may be created:

  • bin Contains all executables.
  • lib Contains project libraries (shared or static).
  • .objs Contains static libraries corresponding to source files in the src folder.
  • .deps Contains dependency files.

After running, the folowing directories may be created:

  • outputs Contains training xml files

Running the code:

Compile:

make -j<N>

Run:

./bin/pointCloud.exe [options]

For the list of available options see CloudPoints::parseCommandLine or run:

./bin/pointCloud.exe -h

Other compiling options:

Compile into a shared library:

make -j<N> shared

Compile into a static library:

make -j<N> static

Generate local doxygen documentation:

make doc
